# 任务：为 RAGAgent 项目实现日志封装

## 背景
RAGAgent 项目需要一个基于 Go 标准库 `log/slog` 的日志模块，该模块必须严格遵循现代日志设计的最佳实践，并具备生产环境所需的特性。

## 角色
资深 Go 后端工程师，熟悉 `log/slog` 的内部机制及可观测性生态。

## 目标
实现一个可复用的 `pkg/logger` 包，提供以下核心功能：

1. **标准化日志接口**
   - 对敏感信息使用 `LogValuer` 接口进行脱敏（例如密码、Token）。
   - 提供 `SensitiveString`、`MaskedJSON` 等辅助类型。

2. **错误日志自动附加堆栈跟踪**
   - 当调用 `Error` 级别的方法时，自动在日志记录中插入 `stacktrace` 字段（使用 `runtime/debug.Stack()`）。
   - 避免重复添加堆栈。

3. **上下文富化**
   - 支持从 `context.Context` 中提取字段（如 `trace_id`、`span_id`、`user_id`）并自动注入每条日志。
   - 提供与 OpenTelemetry 的集成适配点。

4. **灵活的配置选项**
   - 日志级别（`slog.Level`）
   - 输出格式（JSON / Text）
   - 输出目标（`io.Writer`）：`os.Stdout`、文件、或多个目标的组合
   - 是否记录源码位置（`slog.AddSource`）
   - 是否在错误时附加堆栈
   - 采样率（均匀采样，0.0 ~ 1.0）
   - 自定义属性替换函数（例如统一键名为 `snake_case`）

5. **多输出支持**
   - 支持同时写入文件和控制台，通过 `slogmulti.Fanout` 分发日志。

6. **采样**
   - 集成第三方 `slog-sampling` 处理器，在高流量下降低日志量。

7. **本地持久化优先**
   - 封装不应绑定特定的日志管理服务，而是通过标准输出和本地文件解耦，便于使用 Vector / Fluentd 等工具转发。

## 技术要求
- 使用 Go 1.21+ 的 `log/slog`。
- 依赖库：
  - `github.com/samber/slog-multi`
  - `github.com/samber/slog-sampling`
- 所有公开的 API 需保持简洁，核心类型为 `logger.Logger`，它内嵌 `*slog.Logger` 并添加了便捷的上下文方法：
  - `DebugContext(ctx, msg, args...)`
  - `InfoContext(ctx, msg, args...)`
  - `WarnContext(ctx, msg, args...)`
  - `ErrorContext(ctx, err, msg, args...)`
- 提供函数式选项模式（`WithLevel`, `WithFormat`, `WithWriter`, `WithSampling`, `WithContextExtractor`, `WithReplaceAttr`, `WithMultiWriter`, `WithFileOutput` 等）。

## 附加约束
- 代码中应包含合理的注释，解释每个中间件和配置选项的用途。
- 避免在日志记录逻辑中引入不必要的阻塞操作。
- 确保 `stacktraceMiddleware` 仅当级别为 `Error` 时才附加堆栈，且已存在时不重复添加。

## 输出要求

## 示例用法片段（供参考）
```go
log := logger.New(
    logger.WithFormat(logger.FormatJSON),
    logger.WithMultiWriter(
        slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{AddSource: true}),
        slog.NewJSONHandler(mustOpenFile("/var/log/ragagent.log"), nil),
    ),
    logger.WithSampling(0.1),
    logger.WithContextExtractor(otelContextExtractor),
)
log.InfoContext(ctx, "RAGAgent started", "version", "1.0")
if err != nil {
    log.ErrorContext(ctx, err, "init failed")
}
```
