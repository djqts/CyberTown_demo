# 外部前端联动接口

Godot 负责地图渲染、NPC 标记、运镜和动画。聊天、对话、事件流、详情面板由外层前端实现。

---

## 一、Web 嵌入契约

### 1.1 HTML 容器

```html
<div id="cybertown-shell">
  <div id="cybertown-html-loading">加载晨曦镇...</div>
  <canvas id="cybertown-canvas"></canvas>
</div>
```

### 1.2 CSS

```css
#cybertown-shell {
  width: 100%;
  min-height: 720px;
  position: relative;
  background: #18202A;
  overflow: hidden;
}
#cybertown-canvas {
  width: 100%;
  height: 720px;
  display: block;
}
#cybertown-html-loading {
  position: absolute; inset: 0;
  display: grid; place-items: center;
  color: #F3F5F7; background: #18202A;
  z-index: 10;
}
```

### 1.3 配置

Godot 启动时自动读取 `window.CYBERTOWN_CONFIG`，可覆盖默认后端地址：

```javascript
window.CYBERTOWN_CONFIG = {
  httpBaseUrl: "http://localhost:8080",   // HTTP API 地址
  wsUrl:       "ws://localhost:8080/ws",  // WebSocket 地址
  userToken:   "godot-user"               // WebSocket 用户标识
};
```

> 不设置时使用默认值 `localhost:8080`。需在 Godot 引擎加载前定义。

## 二、前端 → Godot（调用 Godot 方法）

所有方法挂载在 `window.cybertownGodot`，通过 `JavaScriptBridge.create_callback` 真实绑定。

### 2.1 地图聚焦

```javascript
// 聚焦地点：缩放至 1.0x + 运镜到建筑中心 + 涟漪高亮
window.cybertownGodot.focusLocation(locationId)
// locationId: int, 1-17

// 聚焦 NPC：运镜到 NPC 所在地点 + 选中该 NPC
window.cybertownGodot.focusNpc(npcId)
// npcId: int, /api/npcs 返回的实际 ID
```

### 2.2 高亮标记

```javascript
// 地点涟漪高亮
window.cybertownGodot.highlightLocation(locationId, duration)
// locationId: int
// duration: float, 可选，默认 1.5s

// 选中 NPC（更新 Godot 内部选中状态，不移动镜头）
window.cybertownGodot.highlightNpc(npcId, duration)
// duration: float, 可选
```

### 2.3 选中操作（仅更新状态，不移动镜头）

```javascript
window.cybertownGodot.selectNpc(npcId)
window.cybertownGodot.selectLocation(locationId)
// 均不触发运镜、不高亮、不发事件通知
```

### 2.4 数据查询

> 返回值均为 **JSON 字符串**，前端需用 `JSON.parse()` 解析。

```javascript
// 地点列表（17 个）
JSON.parse(window.cybertownGodot.getLocationData())
// → Array<{ id, name, icon, district, size, x, y }>

// NPC 列表（15 个）
JSON.parse(window.cybertownGodot.getNpcData())
// → Array<{ id, name, role, location_id, mood, energy, current_goal, gender, age_group, portrait_color }>

// 小镇状态
JSON.parse(window.cybertownGodot.getTownState())
// → { name, day, minute, time, season, weather }

// 某地点的 NPC 列表（用于按钮角标、详情面板在场 NPC）
JSON.parse(window.cybertownGodot.getNpcsAtLocation(locationId))
// → Array<NPC>

// 后端数据在线状态（HTTP 成功 = true，离线 Mock = false）
window.cybertownGodot.getOnlineStatus()
// → bool

// WebSocket 连接状态
window.cybertownGodot.getWsConnected()
// → bool
```

## 三、Godot → 前端（接收 Godot 事件）

### 3.1 注册方式

Godot 同时通过两种方式派发事件，前端按偏好选用：

```javascript
// 方式一：回调函数
window.onGodotEvent = function(event) {
  // event = { type: string, data: object }
};

// 方式二：DOM 事件（React / Vue 友好）
window.addEventListener("godot.map.ready", (e) => {
  // e.type = "godot.map.ready"
  // e.detail = { type: "godot.map.ready", data: {...} }
});
```

### 3.2 事件完整清单

| 事件类型 | 触发时机 | data 内容 |
|---------|---------|----------|
| `godot.map.ready` | 地图加载完成 | `{ online, town_state, locations[], npcs[] }` |
| `godot.npc.selected` | 地图上点击 NPC | `{ npc_id, location_id }` |
| `godot.location.selected` | 地图上点击地点区域 | `{ location_id }` |
| `godot.ws.message` | 收到 WebSocket 消息 | 原始 WS 消息（含 type + data） |
| `godot.ws.connected` | WebSocket 连接成功 | `{}` |
| `godot.ws.disconnected` | WebSocket 断开 | `{}` |
| `godot.offline` | 后端不可用，切换 Mock | `{ reason: "backend_unreachable" }` |
| `godot.validation.failed` | 数据校验发现错误 | `{ ok: false, errors[], warnings[] }` |

### 3.3 godot.map.ready（就绪）

```json
{
  "type": "godot.map.ready",
  "data": {
    "online": true,
    "town_state": { "name": "晨曦镇", "time": "09:15", "season": "春季", "weather": "晴" },
    "locations": [
      { "id": 1, "name": "广场", "icon": "🏛", "district": "central", "size": "large" }
    ],
    "npcs": [
      { "id": 20, "name": "莉娜", "role": "咖啡馆主", "location_id": 2, "mood": "cheerful", "energy": 90, "current_goal": "准备开店", "gender": "女", "age_group": "青年", "portrait_color": "#78B860" }
    ]
  }
}
```

> 外层前端收到后应：隐藏 HTML loading，用 `locations[]` 构建地点按钮列表，用 `npcs[]` 初始化 NPC 面板。

### 3.4 godot.npc.selected（NPC 被点击）

```json
{
  "type": "godot.npc.selected",
  "data": { "npc_id": 20, "location_id": 2 }
}
```

### 3.5 godot.location.selected（地点被点击）

```json
{
  "type": "godot.location.selected",
  "data": { "location_id": 1 }
}
```

### 3.6 godot.ws.message（WebSocket 消息转发）

所有后端 WebSocket 消息，Godot 处理完地图状态后均原封转发给前端：

| 消息类型 | Godot 地图处理 | 转发 |
|----------|-------------|------|
| `npc.replied` | 无 | ✅ |
| `npc.moved` | 更新 NPC 位置 + 路径动画 | ✅ |
| `npc.activity.generated` | 无 | ✅ |
| `npc.idle.action` | 无 | ✅ |
| `npc.mood.changed` | 更新 mood + 闪烁动画 | ✅ |
| `npc.goal.changed` | 更新 current_goal | ✅ |
| `story.event.triggered` | StoryBanner + 涟漪 + 更新 mood/goal | ✅ |
| `npc.interaction.generated` | 更新 mood_changes | ✅ |

转发格式：

```json
{
  "type": "godot.ws.message",
  "data": {
    "type": "npc.replied",
    "data": { "npc_id": 20, "npc_name": "莉娜", "content": "...", "user_token": "web-user" }
  }
}
```

> 对话类消息（replied / activity / idle / interaction）Godot 不展示 UI，前端负责渲染聊天气泡和事件流。

### 3.7 godot.ws.connected / disconnected（连接状态）

```json
{ "type": "godot.ws.connected", "data": {} }
{ "type": "godot.ws.disconnected", "data": {} }
```

> 断线后 3s 自动重连，重连成功再发 `connected`。

### 3.8 godot.offline（离线模式）

```json
{ "type": "godot.offline", "data": { "reason": "backend_unreachable" } }
```

> HTTP 请求超时 5s 后自动切换，地图仍可正常浏览（使用 Mock 数据）。

### 3.9 godot.validation.failed（数据校验）

```json
{
  "type": "godot.validation.failed",
  "data": {
    "ok": false,
    "errors": ["location 7 missing local_map_positions"],
    "warnings": ["npc_sprite_map: 镇长 path not found"]
  }
}
```

> `errors` 为严重问题，`warnings` 为非致命。仅开发环境建议显示，生产环境忽略。

## 四、数据字段速查

### 4.1 Location（地点）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 1-17，与 NPC.location_id 对应 |
| name | string | 中文名称 |
| icon | string | emoji（保留字段，Godot 不显示） |
| district | string | 所属区域 ID |
| size | string | small / medium / large |
| x | int | API 坐标 X（1000 系） |
| y | int | API 坐标 Y（800 系） |

### 4.2 NPC

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 数据库自增，不可写死 |
| name | string | 姓名 |
| role | string | 职业 |
| location_id | int | 当前所在 Location.id |
| mood | string | 英文 key |
| energy | int | 0-100 |
| current_goal | string | 当前目标文本 |
| gender | string | 男 / 女 |
| age_group | string | 儿童 / 青年 / 中年 / 老年 |
| portrait_color | string | 立绘主题色 HEX |

> NPC 详情（personality / catchphrase / appearance）需外层前端自行调用 `GET /api/npcs/{id}` 获取。

## 五、Loading 生命周期

```
网页加载 Godot 引擎
    │ HTML loading 显示 "加载晨曦镇..."
    │
Godot 首帧
    │ LoadingOverlay: "连接后端..."
    │
拉取 /api/town/state + /api/map + /api/npcs
    ├── 成功（< 5s）
    │   │ godot.validation.failed（如有）
    │   │ 连接 WebSocket
    │   │ godot.map.ready  ──→ 前端隐藏 HTML loading
    │   │ godot.ws.connected
    │
    └── 失败或超时（> 5s）
        │ godot.offline  ──→ 前端可显示"离线预览"标识
        │ godot.validation.failed（如有）
        │ godot.map.ready（Mock 数据）
```

## 六、典型联动流程

### 6.1 用户点击外层前端的 "咖啡馆" 按钮

```
1. 前端调用 window.cybertownGodot.focusLocation(2)
2. Godot 缩放至 1.0x + 运镜到咖啡馆 + 涟漪高亮
3. 前端切换面板到咖啡馆详情
```

### 6.2 用户点击地图上的莉娜

```
1. Godot 发出:
   - godot.npc.selected { npc_id: 20, location_id: 2 }
   - window.dispatchEvent(new CustomEvent("godot.npc.selected", ...))
2. 前端收到 → GET /api/npcs/20 → 面板显示莉娜详情
```

### 6.3 后端推送莉娜移动到广场

```
1. Godot 收到 WS: { type: "npc.moved", data: { npc_id: 20, from_location: "咖啡馆", to_location: "广场" } }
2. Godot 内部: 更新莉娜 location_id=1 + BFS 路径动画 + 重排两地 NPC
3. Godot 转发: godot.ws.message { type: "npc.moved", data: {...} }
4. 前端收到 → 更新咖啡馆按钮角标 -1，广场按钮角标 +1
```

### 6.4 后端不可用

```
1. HTTP 请求 > 5s 无响应
2. Godot 发出 godot.offline { reason: "backend_unreachable" }
3. Godot 切换到 Mock 数据，地图仍可正常交互
4. 前端收到 offline → 显示"离线预览"标识
```
